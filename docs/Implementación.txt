Implementación Técnica para Mostrar y Gestionar Lista de Aplicaciones
Para implementar esta funcionalidad en tu aplicación (basada en Python 3.12+, CustomTkinter para UI y ADBController para comandos ADB), seguiré una estructura modular que se integra con tu arquitectura actual. El enfoque es:

Obtener y filtrar la lista de aplicaciones usando comandos ADB.
Mostrar la lista en la UI con separación visual entre apps de sistema y de usuario.
Manejar clics para mostrar detalles y opciones de administración en un panel o popup.

Asumiré que tu ADBController ya tiene un método genérico como execute_adb_command(cmd) que ejecuta comandos vía subprocess y retorna el output. Si no, puedes agregarlo fácilmente.
Paso 1: Obtener y Filtrar la Lista de Aplicaciones

Usa comandos ADB para listar paquetes:
Todas las apps: adb shell pm list packages
Apps de sistema: adb shell pm list packages -s (system apps, preinstaladas).
Apps de usuario: adb shell pm list packages -3 (third-party, instaladas por el usuario).

Parsea el output: Cada línea es package:com.example.app, así que extrae el nombre del paquete quitando "package:".
Almacena en listas/diccionarios para fácil acceso.

Descripción Técnica:

Crea métodos en ADBController para obtener listas filtradas.
Usa subprocess.run para ejecutar comandos, capturando stdout.
Filtra y limpia el output en Python.

Ejemplo de código en ADBController:
Pythonimport subprocess

class ADBController:
    def __init__(self, device_id=None):
        self.device_id = device_id
        self.adb_prefix = ['adb'] + (['-s', device_id] if device_id else [])

    def execute_adb_command(self, cmd):
        full_cmd = self.adb_prefix + cmd
        result = subprocess.run(full_cmd, capture_output=True, text=True, check=True)
        return result.stdout.strip()

    def get_packages(self, filter_type=None):
        cmd = ['shell', 'pm', 'list', 'packages']
        if filter_type == 'system':
            cmd.append('-s')
        elif filter_type == 'user':
            cmd.append('-3')
        output = self.execute_adb_command(cmd)
        packages = [line.split(':')[1] for line in output.splitlines() if line.startswith('package:')]
        return packages

Paso 2: Mostrar la Lista en la UI

Usa CustomTkinter para una interfaz moderna: Un CTkScrollableFrame o CTkListbox para la lista.
Separa visualmente: Dos secciones/tabs (e.g., "Apps de Sistema" y "Apps de Usuario") usando CTkTabview.
Para cada app, muestra un CTkButton o CTkLabel clickable que represente el paquete (puedes obtener nombre amigable con más info en Paso 3).
Actualiza la lista dinámicamente al conectar un dispositivo.

Descripción Técnica:

En tu clase principal de UI, crea un tabview.
Para cada lista (system/user), itera y agrega widgets clickables.
Bind clics a una función que maneje el detalle (ver Paso 3).

Ejemplo de código en tu UI (asumiendo root es tu ventana principal):
Pythonfrom customtkinter import CTkTabview, CTkScrollableFrame, CTkButton

class AppUI:
    def __init__(self, root, adb_controller):
        self.root = root
        self.adb = adb_controller
        self.tabview = CTkTabview(self.root)
        self.tabview.pack(fill='both', expand=True)

        # Tabs para filtrado
        system_tab = self.tabview.add("Apps de Sistema")
        user_tab = self.tabview.add("Apps de Usuario")

        self.populate_list(system_tab, self.adb.get_packages('system'), is_system=True)
        self.populate_list(user_tab, self.adb.get_packages('user'), is_system=False)

    def populate_list(self, tab, packages, is_system):
        scroll_frame = CTkScrollableFrame(tab)
        scroll_frame.pack(fill='both', expand=True)
        for pkg in packages:
            btn = CTkButton(scroll_frame, text=pkg, command=lambda p=pkg: self.show_app_details(p, is_system))
            btn.pack(fill='x', pady=5)

Nota: Para nombres amigables (no solo paquete), en populate_list, obtén el label con adb shell pm dump {pkg} | grep 'label=' o integra en Paso 3.

Paso 3: Mostrar Detalles y Opciones al Hacer Clic

Al clic, usa adb shell dumpsys package {package_name} para info detallada:
Nombre, versión, UID: Parsea secciones como "versionName=", "userId=".
Permisos: Sección "requested permissions:".
Ruta APK: "codePath=".
Fecha instalación: "firstInstallTime=" (timestamp, convierte a fecha con datetime).
Tipo: Ya sabes si es system/user del filtro.

Muestra en un popup (CTkToplevel) o panel lateral con labels para info y buttons para acciones (desinstalar, extraer, mirroring, limpiar, desactivar).
Para acciones, usa tus métodos existentes en ADBController (e.g., uninstall(package), extract_apk(package), etc.).

Descripción Técnica:

Parsea el output de dumpsys con regex o splitting lines.
Crea una clase para el detalle o una función que genere el popup.

Ejemplo de código:
Pythonimport re
from datetime import datetime
from customtkinter import CTkToplevel, CTkLabel, CTkButton

# En ADBController, agrega:
def get_app_details(self, package):
    output = self.execute_adb_command(['shell', 'dumpsys', 'package', package])
    details = {}
    details['name'] = re.search(r'versionName=(.*)', output).group(1) if re.search(r'versionName=(.*)', output) else 'N/A'
    details['version'] = re.search(r'versionCode=(.*)', output).group(1) if re.search(r'versionCode=(.*)', output) else 'N/A'
    details['uid'] = re.search(r'userId=(.*)', output).group(1) if re.search(r'userId=(.*)', output) else 'N/A'
    details['apk_path'] = re.search(r'codePath=(.*)', output).group(1) if re.search(r'codePath=(.*)', output) else 'N/A'
    install_time = re.search(r'firstInstallTime=(.*)', output).group(1)
    details['install_date'] = datetime.fromtimestamp(int(install_time)/1000).strftime('%Y-%m-%d %H:%M:%S') if install_time else 'N/A'
    perms_section = re.search(r'requested permissions:(.*?)install permissions:', output, re.DOTALL)
    details['permissions'] = [p.strip() for p in perms_section.group(1).splitlines() if p.strip()] if perms_section else []
    return details

# En AppUI, la función de clic:
def show_app_details(self, package, is_system):
    details = self.adb.get_app_details(package)
    popup = CTkToplevel(self.root)
    popup.title(f"Detalles de {package}")

    # Mostrar info
    CTkLabel(popup, text=f"Nombre: {details['name']}").pack()
    CTkLabel(popup, text=f"Versión: {details['version']}").pack()
    CTkLabel(popup, text=f"UID: {details['uid']}").pack()
    CTkLabel(popup, text=f"Ruta APK: {details['apk_path']}").pack()
    CTkLabel(popup, text=f"Fecha Instalación: {details['install_date']}").pack()
    CTkLabel(popup, text="Permisos:").pack()
    for perm in details['permissions']:
        CTkLabel(popup, text=perm).pack()

    # Opciones de administración
    CTkButton(popup, text="Desinstalar", command=lambda: self.adb.uninstall(package)).pack()  # Asume método uninstall

    CTkButton(popup, text="Limpiar Datos", command=lambda: self.adb.clear_data(package)).pack()
    CTkButton(popup, text="Desactivar", command=lambda: self.adb.disable(package) if is_system else None).pack()  # Solo para system/bloatware

Consideraciones Adicionales

Manejo de Errores: Envuelve llamadas a subprocess en try-except para manejar dispositivos desconectados (e.g., subprocess.CalledProcessError).
Actualización en Tiempo Real: Usa un hilo (threading.Thread) para refrescar la lista cada X segundos si detectas cambios.
Seguridad: Para apps de sistema, deshabilita opciones como desinstalar (usa pm disable en su lugar).
Optimización: Si la lista es larga, usa paginación o búsqueda en la UI (agrega CTkEntry para filtrar por nombre).
Integración con Existente: Llama a AppUI en tu init principal después de conectar el dispositivo. Asegúrate que device_id se pase correctamente si usas multi-device.

Esto se alinea con tu diseño modular: La lógica ADB en controller, UI separada. Prueba con un emulador o device real para depurar parses (dumpsys output varía por Android version).
